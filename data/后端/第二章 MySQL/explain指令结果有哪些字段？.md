explain是查看SQL的执行计划，有以下几大核心字段：
- possible_keys 可能被用到的索引
- key 实际使用的索引，NULL表示没有使用索引
- key_len 索引的长度
- rows 扫描的数据行数
- type 数据扫描的类型，这个是最核心的字段，直接决定SQL性能

type从性能高到低排序：
- system 当前查询的表是系统表，已经加载到内存，性能最高
- const 使用了主键或者唯一索引与常量值进行比较
- eq_ref 多表连接查询中使用了主键或唯一索引
- ref 使用了非唯一索引，或者是唯一索引的非唯一性前缀
- range 使用了范围查询
- index 使用了索引，但仍然进行了全表扫描，由于无需排序，所以性能比All好
- All 全表扫描
