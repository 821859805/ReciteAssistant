Python 没有严格的访问控制（如 Java 的 private/protected），而是依赖**命名约定**。

**命名约定：**
```python
class Account:
    def __init__(self, owner, balance):
        self.owner = owner          # 公有属性
        self._balance = balance     # 约定私有（单下划线）
        self.__id = id(self)        # 名称修饰（双下划线）

a = Account("Alice", 1000)
a.owner       # "Alice"（正常访问）
a._balance    # 1000（可以访问，但约定不应该）
# a.__id      # AttributeError!
a._Account__id  # 可以通过名称修饰后的名字访问
```

**单下划线 `_`（约定私有）：**
- 纯约定，不强制
- 告诉其他开发者"这是内部实现，请勿直接访问"
- `from module import *` 不会导入以 `_` 开头的名字

**双下划线 `__`（名称修饰 Name Mangling）：**
- Python 会将 `__attr` 改为 `_ClassName__attr`
- 目的是避免子类意外覆盖父类的属性
- 不是为了安全性，只是为了避免命名冲突

**property 装饰器（属性访问控制）：**
```python
class Temperature:
    def __init__(self, celsius):
        self._celsius = celsius
    
    @property
    def celsius(self):
        """getter"""
        return self._celsius
    
    @celsius.setter
    def celsius(self, value):
        """setter"""
        if value < -273.15:
            raise ValueError("温度不能低于绝对零度")
        self._celsius = value
    
    @property
    def fahrenheit(self):
        """只读属性（没有 setter）"""
        return self._celsius * 9 / 5 + 32

t = Temperature(25)
print(t.celsius)       # 25（像属性一样访问，实际调用 getter）
t.celsius = 30         # 调用 setter
print(t.fahrenheit)    # 86.0
# t.fahrenheit = 100   # AttributeError!（只读）
```
