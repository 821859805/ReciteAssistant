**抽象类**是不能被实例化的类，用于定义子类必须实现的接口。

```python
from abc import ABC, abstractmethod

class Animal(ABC):
    def __init__(self, name):
        self.name = name
    
    @abstractmethod
    def speak(self):
        """子类必须实现此方法"""
        pass
    
    @abstractmethod
    def move(self):
        pass
    
    # 非抽象方法（可以有具体实现）
    def describe(self):
        return f"我是 {self.name}"

# animal = Animal("test")  # TypeError! 不能实例化抽象类

class Dog(Animal):
    def speak(self):          # 必须实现所有抽象方法
        return "汪汪"
    
    def move(self):
        return "跑"

dog = Dog("旺财")
dog.speak()     # "汪汪"
dog.describe()  # "我是 旺财"
```

**抽象属性：**
```python
class Shape(ABC):
    @property
    @abstractmethod
    def area(self):
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    @property
    def area(self):
        return 3.14159 * self.radius ** 2
```

**抽象类方法与静态方法：**
```python
class Base(ABC):
    @classmethod
    @abstractmethod
    def create(cls):
        pass
    
    @staticmethod
    @abstractmethod
    def validate(data):
        pass
```

**Python 接口的惯用方式：**
Python 没有像 Java 那样的 `interface` 关键字，接口通常通过以下方式实现：
1. **抽象类（ABC）**：正式定义接口
2. **Protocol（typing.Protocol）**：结构化子类型（鸭子类型的形式化）
3. **约定**：文档中说明需要实现的方法

**ABC vs Protocol：**
| 特性 | ABC | Protocol |
|------|-----|----------|
| 需要继承 | 是 | 否 |
| 运行时检查 | isinstance() 可用 | 需要 runtime_checkable |
| 适用场景 | 传统 OOP | 鸭子类型 |
