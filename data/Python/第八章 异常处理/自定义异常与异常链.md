**自定义异常：**
```python
class AppError(Exception):
    """应用基础异常"""
    pass

class ValidationError(AppError):
    """数据验证错误"""
    def __init__(self, field, message):
        self.field = field
        self.message = message
        super().__init__(f"{field}: {message}")

class NotFoundError(AppError):
    """资源未找到"""
    def __init__(self, resource, id):
        self.resource = resource
        self.id = id
        super().__init__(f"{resource}(id={id}) 未找到")

# 使用
try:
    raise ValidationError("email", "格式不正确")
except ValidationError as e:
    print(e.field)     # "email"
    print(e.message)   # "格式不正确"
    print(e)           # "email: 格式不正确"
```

**异常链（Exception Chaining）：**
```python
# raise ... from ...（显式异常链）
try:
    result = int("abc")
except ValueError as e:
    raise RuntimeError("数据处理失败") from e
# RuntimeError: 数据处理失败
#   上述异常是下列异常的直接原因:
# ValueError: invalid literal for int() with base 10: 'abc'

# 隐式异常链（在 except 中抛出新异常）
try:
    result = int("abc")
except ValueError:
    raise RuntimeError("处理失败")
# 也会保留原始异常信息

# 禁用异常链
try:
    result = int("abc")
except ValueError:
    raise RuntimeError("处理失败") from None
# 不显示原始异常
```

**断言 assert：**
```python
# 用于开发时的调试检查
assert condition, "错误信息"
# 等价于:
# if not condition:
#     raise AssertionError("错误信息")

def divide(a, b):
    assert b != 0, "除数不能为零"
    return a / b

# 注意：python -O（优化模式）会忽略所有 assert
# 因此不应该在 assert 中放重要的逻辑
```

**异常处理最佳实践：**
- 捕获具体的异常类型，而非笼统的 `Exception`
- 不要用空的 `except:` 或 `except Exception:`（会吞掉所有错误）
- 使用 `finally` 确保资源释放
- 自定义异常应继承自 `Exception`，而非 `BaseException`
- 在合适的层级处理异常（不一定在发生处）
